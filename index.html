🌩️ Dominio 1: Conceptos de la Nube (24%)

¿Qué es la Computación en la Nube?
La nube es la disponibilización de recursos de TI a través de Internet bajo demanda. En lugar de comprar servidores físicos, rentas capacidad cuando la necesitas.
💡 Analogía: Como contratar electricidad - pagas solo lo que consumes, no necesitas tener tu propia planta eléctrica.

Modelos de Servicio en la Nube

IaaS
Infraestructura como Servicio
→
PaaS
Plataforma como Servicio
→
SaaS
Software como Servicio
Modelo Descripción Ejemplo AWS Analogía
IaaS Solo infraestructura, tú instalas todo Amazon EC2 Departamento vacío para amueblar
PaaS Plataforma lista, solo despliegas tu app AWS Elastic Beanstalk Departamento amueblado
SaaS Aplicación completa lista para usar Amazon WorkSpaces Suite de hotel todo incluido
💡 Beneficios Clave de la Nube:
• Elasticidad: Escala automáticamente
• Economía: Paga solo lo que usas
• Agilidad: Recursos en minutos
• Alcance Global: Deploy mundial
• Confiabilidad: Infraestructura redundante

🔒 Dominio 2: Seguridad y Cumplimiento (30%)

Modelo de Responsabilidad Compartida
AWS es responsable de la "seguridad DE la nube" (infraestructura física, hardware, red). El cliente es responsable de la "seguridad EN la nube" (datos, configuraciones, usuarios).
🏢 Analogía: AWS mantiene seguro el edificio (puertas, alarmas), tú aseguras tu apartamento (cerrar con llave, no dejar ventanas abiertas).

Responsabilidades de Seguridad

AWS
Seguridad DE la nube

• Data centers físicos
• Hardware y red
• Hipervisores
• Servicios administrados

Cliente
Seguridad EN la nube

• Datos y cifrado
• Configuración IAM
• Sistema operativo
• Security Groups

AWS IAM (Identity and Access Management)
Servicio para gestionar usuarios, grupos, roles y permisos. Permite controlar quién puede acceder a qué recursos.
⭐ Mejores Prácticas:
• Activar MFA en cuenta root
• Crear usuarios IAM individuales
• Principio de menor privilegio
• Rotar credenciales regularmente

💰 Dominio 4: Facturación y Soporte (12%)

Modelos de Precios AWS
AWS opera bajo el principio "paga por lo que usas" con diferentes opciones para optimizar costos.
🎯 Factores de Costo: Cómputo + Almacenamiento + Transferencia de datos
Modelo de Compra Descripción Ahorro Cuándo Usar
On-Demand Pago por hora/segundo sin compromiso 0% (precio base) Cargas impredecibles, desarrollo
Reserved Instances Compromiso 1-3 años Hasta 75% Uso constante conocido
Spot Instances Capacidad no utilizada Hasta 90% Cargas tolerantes a interrupciones
Savings Plans Compromiso de gasto $/hora Hasta 72% Flexibilidad entre servicios

🎧 Planes de Soporte AWS

Plan Costo Características Clave Para Quién
Basic Gratis Documentación, foros Todos los usuarios
Developer $29/mes o 3% Email soporte, horario oficina Desarrollo/pruebas
Business $100/mes o 10-3% Soporte 24/7, Trusted Advisor Cargas de producción
Enterprise $15,000/mes o 10% TAM dedicado, respuesta <15min Empresas críticas
💡 Consejos para Optimizar Costos:
• Usar Reserved Instances para cargas estables
• Implementar Auto Scaling
• Apagar recursos no utilizados
• Elegir la región correcta
• Monitorear con Cost Explorer y Budgets
